# 系统架构文档

> 本文档记录系统**当前架构状态**，由 harness 自动维护。
> 每次需求设计完成后更新（plan 阶段），每次编码完成后校准（Phase 2 复盘阶段）。
>
> **重要**：本文档描述的是系统"现在是什么样"，而非"应该是什么样"。
> 如果发现文档与代码不一致，以代码为准，并更新文档。

- 更新来源: {需求名} / Phase 2 校准
